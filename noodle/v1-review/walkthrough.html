Aura v1 — Design Review

Walk through each screen. Click to mark reviewed. Use dev panel to navigate.

Before you start

Open index.html in the browser next to this. Clear localStorage (dev panel → Reset). Each step tells you which dev panel button to click.

1
Welcome Screen
tap to mark done
Dev panel → Welcome
What changed: Entity gate removed. New users land here directly — no "Do you have a biological body?" gate. Entity silently defaults to human.
  • ◆ Does the welcome screen feel like a strong first impression?
  • ◆ Is "Let's Go" vs "Sign In" the right hierarchy?
  • ◆ Aura visualization — does it look good with zero data?

2
Onboarding (5 Questions)
tap to mark done
Dev panel → Onboard
What changed: Trimmed from 11 to 5 questions. Removed: entity gate Q, self-driving, hot dog, Mars, coffee, water wet. Kept the fun opinionated ones that build momentum.
  • ◆ Do 5 questions feel like enough to build momentum?
  • ◆ Is the question order right? (chicken/egg → pizza → superpower → dogs/cats → AI jobs)
  • ◆ Does it feel fast and fun, or are any questions filler?
  • ◆ Last Q "Will AI replace most jobs?" — does it prime the AI differentiator well?
↓
3
Quick Profile (Auto-Start)
tap to mark done
Complete onboarding → auto-starts (or dev panel → Assess Q)
What changed: Path choice screen removed. After onboarding, Quick Profile starts immediately. No "Map Your Personality" vs "Explore Questions" fork. Just straight into the assessment.
  • ◆ Does the transition from onboarding to assessment feel seamless?
  • ◆ Is skipping path choice the right call, or does the user need a breath?
  • ◆ Does the assessment question UI feel different enough from onboarding?
↓
4
Quick Profile Results
tap to mark done
Complete Quick Profile → results (or dev panel → Results)
What changed: Enhanced from a simple text card to: mini aura visualization + 5 trait spectrum bars + personality snapshot text + share button (generates compare link) + "Keep going" CTA. Previously went straight to the picker.
  • ◆ Is the aura viz meaningful with only Quick Profile data?
  • ◆ Do the trait bars communicate clearly? Labels readable?
  • ◆ Is the personality snapshot text accurate and compelling?
  • ◆ Share button — is it prominent enough for the viral moment?
  • ◆ "Keep going" vs "Browse all assessments" — right hierarchy?
↓
5
Assessment Picker (Unlocked)
tap to mark done
Dev panel → Assess
What changed: No changes to the picker itself — but this is now the first time users see it (after completing Quick Profile). The tier system already gates categories.
  • ◆ After QP results, does the picker feel like a natural next destination?
  • ◆ Is the unlocked vs locked distinction clear?
— bonus screens —
6
AI Entry Path
tap to mark done
Load with ?entity=ai URL param
What changed: AIs no longer go through the entity gate UI. Instead, ?entity=ai URL param silently sets entity to AI, stores it, cleans the URL, and proceeds to welcome. The AI differentiator is now a clean URL — shareable, no friction.
  • ◆ Is a URL param the right entry for AIs, or do they need UI?
  • ◆ After AI entry, does the welcome screen acknowledge them?
  • ◆ Entity gate still reachable via dev panel — should it stay for manual testing?
7
Progressive Disclosure
tap to mark done
Dev panel → Home (categories screen). Try with 0, 1, 3, 5 assessments.
What changed: Features now gate behind completion milestones.
• Tier 0 (new): Hero assessment only, minimal nav
• Tier 1 (1+ done): Full picker, profile
• Tier 2 (3+ done): QotD, community browse
• Tier 3 (5+ or logged in): Verify, stamps, history
  • ◆ Are the tier thresholds right? (1 / 3 / 5)
  • ◆ Does the categories screen feel too empty at tier 0?
  • ◆ When features unlock, is there any indication, or do they just appear?
8
Compare Auras
tap to mark done
Dev panel → Compare (or share link from QP results)
What changed: New screen. Share button on QP results encodes traits in a ?compare= URL. Recipients see sender's trait bars; if they've taken QP, side-by-side overlay. If not, CTA to take the assessment.
  • ◆ Does side-by-side comparison read clearly?
  • ◆ Is the "take the assessment to compare" CTA compelling?
  • ◆ Is this the right viral mechanic for the demo?
